ACCESO REMOTO

El Desafío

Las empresas se encuentran bajo una constante presión por asegurar ingresos, administrar los gastos y mejorar la productividad de sus empleados. Para hacer frente a estos desafíos deben valerse de la tercerización, la asociación con partners y una fuerza de trabajo flexible y móvil.

Como resultado, las nuevas demandas para el negocio se traducen en:

  • - Creciente necesidad de los usuarios de acceder a los recursos internos, desde cualquier lugar en el que se encuentren
  • - Creciente variedad de dispositivos y tecnologías para acceder a los recursos internos
  • - Soporte a un número cada vez mayor de aplicaciones, como ser CRM, ERP, voz y video, entre otras

La “empresa extendida” de empleados, socios y consultores trabajando en forma remota y móvil, realmente presenta beneficios en la productividad. Pero ha dejado a las organizaciones con redes corporativas sin límites definidos, planteando el desafío de responder a las necesidades de conectividad de los usuarios sin perder de vista aspectos de seguridad.

La Solución

La solución de HYNET ofrece, a usuarios remotos - teletrabajadores, usuarios móviles, clientes y asociados de negocio-, la posibilidad de acceder y utilizar en forma segura aplicaciones corporativas como si estuvieran en la oficina.

SSL VPN surge del concepto de red privada virtual, es decir, la práctica de utilizar una red pública como Internet para transmitir datos privados. Originariamente esta práctica se implementaba mediante el cifrado de los datos a través de algún tipo de transporte a nivel red, como el conocido IPSec, requiriendo que ambas puntas contaran con software especialmente configurado para tal fin.

SSL VPN utiliza una metodología diferente. En lugar de confiar en que el usuario final tenga configurado un software cliente en su equipo, SSL VPN utiliza SSL/HTTPS, el mecanismo de transporte seguro que ya viene incluido en todos los Web browsers estándar, sin necesidad de software o configuración adicional.

Las ventajas clave de esta solución incluyen, no sólo la facilidad de mantenimiento, sino también alto nivel de seguridad y alta disponibilidad. Ello se logra a través de:

Control del acceso. Acceso a todo tipo de aplicaciones de manera previsible, confiable y encriptada para todos los usuarios, logrando la máxima productividad.

Control del uso. Para determinar qué uso está permitido y denegar el resto, asegurando conformidad con requerimientos internos y regulaciones externas en cuanto a seguridad de datos.

Control de las amenazas. Para detectar y remover contenido malicioso en múltiples niveles, evitando comprometer los datos internos y perder productividad.

Ventajas y Beneficios

Al utilizar SSL VPN, la conexión entre el usuario móvil y el recurso interno se realiza a través de una conexión Web a nivel aplicación, en contraposición al “tunel” que se abre con las VPNs via IPSec, a nivel red. El uso de SSL es ideal para el usuario móvil porque:

  • - SSL VPNs no requieren la descarga de un cliente en el dispositivo utilizado para acceder los recursos corporativos
  • - SSL VPNs no necesitan ser configurados por el usuario final
  • - SSL está disponible siempre que tenga instalado un Web browser estándar
  • - SSL opera a nivel aplicación, independientemente de cualquier sistema operativo, con lo cual cualquier actualización del SO no afecta la implementación de SSL
  • - Al operar a nivel aplicación, es posible ofrecer control de acceso a las aplicaciones con alta granularidad, siendo ideal para trabajadores móviles y aquellos que se conectan desde puntos que están fuera del control de la compañía (untrusted)

Muchas empresas ya cuentan con alguna forma de conectividad para acceso remoto o vía extranet a los recursos internos. Las redes privadas virtuales (VPNs) basadas en una conexión IP y el protocolo IPSec, resuelven eficazmente las necesidades de conexión de sitio a sitio. Sin embargo, para usuarios móviles, presentan importantes inconvenientes y altos costos de mantenimiento. Para socios de negocio o clientes, resultan imposibles de implementar ya que requieren software instalado y configurado en cada dispositivo y provee acceso completo a la red, exponiendo todos los recursos corporativos. Como alternativa, algunas compañías eligieron implementar una extranet para sus socios y clientes, una solución que resulta cara y compleja de liberar y mantener.
